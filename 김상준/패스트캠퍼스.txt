Chapter 2. 자료구조와 알고리즘
	1. 자료구조의 개요
		1) 기본적 자료구조들
			- 선형 구조: 배열, 연결 리스트, 스택, 큐
			- 비선형 구조: 트리, 그래프
		2) 프로그램 성능 측정 방법론
			- 시간 복잡도: 알고리즘에 사용되는 연산 횟수 -> big-O 표기법을 사용
			- 공간 복잡도: 알고리즘에 사용되는 메모리의 양 -> MB단위로 표기
	
	2. 연결 리스트
		1) 배열 기반 리스트 특징
			- 특정한 위치의 원소에 즉시 접근가능
			- 데이터가 들어갈 공간을 메모리에 미리 할당해야함
			- 원하는 위치로 삽입, 삭제가 비효율적
		2) 연결 리스트 특징
			- 포인터와 구조체를 사용하여 구현
			- 중간 지점에 노드를 추가하거나 삭제
			- 필요할 때 메모리 공간을 할당
			- 삽입, 삭제가 배열에 비해서 간단
			- 인덱스로 즉시 접근 불가
		3) 단일 연결 리스트
		4) 양방향 연결리스트
	
	3. 스택
		1) Last In First Out -> 배열, 연결리스트를 통해 구현
		2) 계산기 구현
			- 중위 표기법: 일반적으로 사람이 표기하는 수식
			- 후위 표기법: 컴퓨터가 계산하기에 편한 수식, 연산자가 뒤에
			- 중위 -> 후위 전환 방법
				-> 1. 피연산자는 바로 출력
				-> 2. 연산자가 들어오면 우선순위가 높거나 같은 것을 빼고 자신을 스택에 포함
				-> 3. '(' 를 만나면 무조건 스택에 담음
				-> 4. ')' 를 만나면 '(' 을 만날때까지 스택에서 출력
			- 후위식 계산
				-> 1. 피연산자를 만나면 스택
				-> 2. 연산자를 만나면 스택에서 두개의 피연산자를 꺼내 연산한 뒤 스택에 담음
	
	4. 큐
		1) First In First Out -> 배열, 연결리스트를 통해 구현
